# Push 등록 기기 관리 운영 가이드

## 저장 구조와 호환성

`users/{uid}/alertSettings/default` 문서에는 다음 두 필드가 함께 존재할 수 있습니다.

- `pushTokens: string[]`: 기존 사용자와 이전 엔진 호환을 위한 토큰 배열
- `pushDevices: PushDevice[]`: 앱 생성 등록 ID, 토큰, 정규화된 플랫폼·브라우저·기기 유형, 등록/최근 확인/토큰 갱신 시각

신규 등록과 삭제는 Firestore transaction에서 두 배열을 함께 갱신합니다. 기존 `pushTokens`만 있는 문서는 마이그레이션 없이 계속 발송 대상이며, 설정 화면에는 **기존 등록 기기**로 표시됩니다. Python 엔진은 `pushDevices[].token`을 먼저 읽고 `pushTokens`를 합친 뒤 중복을 제거합니다.

등록 ID는 브라우저의 사용자별 localStorage에 보관합니다. 저장소 삭제, 새 PWA 컨텍스트 또는 브라우저 재설치 후에는 새 기기 등록으로 나타날 수 있습니다. FCM 토큰은 UI 식별자나 문서 ID로 쓰지 않으며 전체 값은 화면·로그·문서에 노출하지 않습니다.

## 기기 정보와 최근 확인의 의미

브라우저가 제공하는 정보에서 Android/Windows/macOS/iOS/Linux, Chrome/삼성 인터넷/Edge/Safari/Firefox, 모바일/태블릿/데스크톱만 정규화합니다. 실제 하드웨어 모델명, 전체 User-Agent, IP, 위치는 저장하지 않습니다.

`lastSeenAt`의 UI 표시는 **최근 확인**입니다. Push 등록 성공 또는 설정 화면에서 기존 등록 ID와 현재 토큰을 정확히 확인했을 때만 갱신되며, 반복 쓰기는 24시간 간격으로 제한됩니다. 일반 페이지 방문 시각이나 실제 마지막 접속 시각을 뜻하지 않습니다. 오래된 항목은 자동 삭제하지 않습니다.

## 삭제와 전체 초기화

- 개별/복수 삭제는 선택한 기기 ID 또는 정확한 레거시 토큰만 transaction으로 제거합니다. 배열 마지막 항목을 임의로 삭제하는 fallback은 없습니다.
- 현재 기기를 삭제한 경우에만 해당 브라우저에서 Firebase `deleteToken()`을 호출하고 로컬 등록 ID를 지웁니다.
- 다른 기기는 원격 `deleteToken()`이 불가능하므로 Firestore 발송 대상에서만 제거합니다.
- **기기 알림 등록 초기화**는 서버의 `pushTokens`와 `pushDevices`를 모두 비운 후 현재 브라우저의 로컬 토큰 삭제를 시도합니다. 서버 초기화 성공과 로컬 정리 실패는 사용자에게 구분해 안내합니다.
- FCM이 영구 `UNREGISTERED`로 확인한 토큰은 Python 엔진이 두 구조에서 함께 제거합니다. 일시 오류와 `INVALID_ARGUMENT`, `SENDER_ID_MISMATCH`만으로는 삭제하지 않습니다.

## 배포와 수동 검증

1. `firestore.rules`를 배포해 소유자 제한, 설정 필드 타입, 최대 100개 등록 제한을 적용합니다.
2. Android Chrome, 삼성 인터넷, 데스크톱 브라우저에서 각각 등록하고 별도 목록 항목인지 확인합니다.
3. 같은 브라우저 재등록 후 항목이 중복되지 않고 최근 확인/토큰 갱신이 유지되는지 확인합니다.
4. 현재 기기, 다른 기기, 복수 기기, 레거시 항목 삭제를 각각 검증합니다.
5. 전체 초기화 후 0대 표시와 현재 기기 재등록을 확인합니다.
6. **테스트 Push**, **테스트 Telegram**, **둘 다 테스트**를 실행해 실제 엔진 결과와 수신을 확인합니다.
7. 로그아웃·로그인 후 목록 유지와 다른 계정 데이터 접근 차단을 확인합니다.

실기기 검증 결과에는 브라우저/OS, 등록, 목록 표시, 현재 기기 판별, 개별 삭제, 전체 초기화, 초기화 후 Push 재수신을 각각 기록합니다. Preview에 Firebase 환경 변수가 없으면 Push 성공으로 기록하지 않습니다.
